# Avatar 头像 [\#](https://chatui.io/components/avatar#Avatar%20%E5%A4%B4%E5%83%8F)

## 代码演示 [\#](https://chatui.io/components/avatar#%E4%BB%A3%E7%A0%81%E6%BC%94%E7%A4%BA)

```demo-code jsx
import React from 'react';
import { Avatar } from '@chatui/core';

export default function() {
  const img = '//img.alicdn.com/imgextra/i2/O1CN01fPEB9P1ylYWgaDuVR_!!6000000006619-0-tps-132-132.jpg';

  return (
    <div>

      <Avatar src={img} size="sm" />
      <Avatar src={img} />
      <Avatar src={img} size="lg" />

      <h3>头像形状</h3>
      <Avatar src={img} size="sm" shape="square" />
      <Avatar src={img} shape="square" />
      <Avatar src={img} size="lg" shape="square" />
    </div>
  );
}
```

## Props [\#](https://chatui.io/components/avatar#Props)

### Avatar [\#](https://chatui.io/components/avatar#Avatar)

| 参数  | 类型                 | 默认值   | 说明                     |
| ----- | -------------------- | -------- | ------------------------ |
| src   | string               |          | 图像的资源地址           |
| url   | string               |          | 头像链接                 |
| alt   | string               |          | 图像无法显示时的替代文本 |
| size  | `sm` \| `md` \| `lg` |          | 头像的大小               |
| shape | `circle` \| `square` | `circle` | 头像的形状               |

# Chat 对话容器 [\#](https://chatui.io/components/chat#Chat%20%E5%AF%B9%E8%AF%9D%E5%AE%B9%E5%99%A8)

## 代码演示 [\#](https://chatui.io/components/chat#%E4%BB%A3%E7%A0%81%E6%BC%94%E7%A4%BA)

```demo-code jsx
import React from 'react';
import Chat, { Bubble, useMessages } from '@chatui/core';

const initialMessages = [\
  {\
    type: 'system',\
    content: { text: '88VIP专属智能客服小蜜 为您服务' },\
  },\
  {\
    type: 'text',\
    content: { text: 'Hi，我是你的专属智能助理小蜜，有问题请随时找我哦~' },\
    user: {\
      avatar: 'https://gw.alicdn.com/imgextra/i2/O1CN01fPEB9P1ylYWgaDuVR_!!6000000006619-0-tps-132-132.jpg',\
    },\
  },\
];

// 默认快捷短语，可选
const defaultQuickReplies = [\
  {\
    icon: 'message',\
    name: '联系人工服务',\
    isNew: true,\
    isHighlight: true,\
  },\
  {\
    name: '短语1',\
    isNew: true,\
  },\
  {\
    name: '短语2',\
    isHighlight: true,\
  },\
  {\
    name: '短语3',\
  },\
];

export default function() {
  // 消息列表
  const { messages, appendMsg } = useMessages(initialMessages);

  // 发送回调
  function handleSend(type, val) {
    if (type === 'text' && val.trim()) {
      appendMsg({
        type: 'text',
        content: { text: val },
        position: 'right',
      });

      // TODO: 发送请求
      // 模拟回复消息
      setTimeout(() => {
        appendMsg({
          type: 'text',
          content: { text: '亲，您遇到什么问题啦？请简要描述您的问题~' },
        });
      }, 1000);
    }
  }

  // 快捷短语回调，可根据 item 数据做出不同的操作，这里以发送文本消息为例
  function handleQuickReplyClick(item) {
    handleSend('text', item.name);
  }

  function renderMessageContent(msg) {
    const { type, content } = msg;

    // 根据消息类型来渲染
    switch (type) {
      case 'text':
        return <Bubble content={content.text} />;
      case 'image':
        return (
          <Bubble type="image">
            <img src={content.picUrl} alt="" />
          </Bubble>
        );
      default:
        return null;
    }
  }

  return (
    <Chat
      navbar={{ title: '智能助理' }}
      messages={messages}
      renderMessageContent={renderMessageContent}
      quickReplies={defaultQuickReplies}
      onQuickReplyClick={handleQuickReplyClick}
      onSend={handleSend}
    />
  );
}
```

```demo-code jsx
import React, { useState } from 'react';
import { Checkbox, CheckboxGroup } from '@chatui/core';

const options = [\
  { label: 'Apple', value: 'Apple' },\
  { label: 'Pear', value: 'Pear', disabled:true },\
  { label: 'Orange', value: 'Orange' },\
  { label: 'Banana', value: 'Banana' },\
];

export default function () {
  const [value1, setValue] = useState(['Apple']);
  const [value2, setValue2] = useState([]);
  const [checked, setChecked] = useState(false);

  function handleChange(val) {
    setValue(val);
  }

  function handleChange2(val) {
    setValue2(val);
  }

  return (
    <div>
      <h3>基础用法</h3>
      <Checkbox
        value="abc"
        label="ABC"
        checked={checked}
        onChange={() => setChecked(!checked)}
      />

      <h3>禁用状态</h3>
      <Checkbox value="abc" label="ABC" disabled />

      <h3>复选框组</h3>
      <CheckboxGroup value={value1} options={options} onChange={handleChange} />

      <h3>块级显示</h3>
      <CheckboxGroup
        value={value2}
        options={options}
        onChange={handleChange2}
        block
      />
    </div>
  );
}
```

在没有数据或加载失败的情况下，向用户反馈的状态，可以自定义展示的类型状态。

## 代码演示 [\#](https://chatui.io/components/empty#%E4%BB%A3%E7%A0%81%E6%BC%94%E7%A4%BA)

```demo-code jsx
import React from 'react';
import { Empty, Button } from '@chatui/core';

export default function() {
  return (
    <div>
      <h3>基础用法</h3>
      <Empty tip="暂无数据" />

      <h3>定制内容</h3>
      <Empty type="error" tip="oops..加载失败，点击重试">
        <Button color="primary">刷新</Button>
      </Empty>
    </div>
  );
}
```

| type     | string    |        | 类型，影响显示图片                |
| image    | string    |        | 自定义显示图片，优先级高于 `type` |
| tip      | string    |        | 自定义描述内容                    |
| children | ReactNode |        |                                   |

# Icon 图标 [\#](https://chatui.io/components/icon#Icon%20%E5%9B%BE%E6%A0%87)

通过 SVG Symbol 实现的矢量图形。使用图标你需要安装 ChatUI 图标组件包：
`<script src="//g.alicdn.com/chatui/icons/2.6.2/index.js"></script>`

## 代码演示 [\#](https://chatui.io/components/icon#%E4%BB%A3%E7%A0%81%E6%BC%94%E7%A4%BA)

```demo-code jsx
import React from 'react';
import { Icon } from '@chatui/core';

const symbols = document.getElementById('__CHATUI_ICONS__').querySelectorAll('symbol');

export default function() {
  return (
    <div>
      <h3>基础用法</h3>
      <Icon type="bullhorn" />

      <h3>旋转动画</h3>
      <Icon type="spinner" spin />

      <h3>图标列表</h3>
      <div className="icon-list">
        {Array.from(symbols).map((item) => {
          const icon = item.id.replace('icon-', '');
          return (
            <div className="icon-box">
              <Icon type={icon} />
              <span>{icon}</span>
            </div>
          )
        })}
      </div>
    </div>
  );
}
```

```demo-code jsx
import React, { useState } from 'react';
import { Input } from '@chatui/core';

export default function() {
  const [value1, setValue1] = useState('');
  const [value2, setValue2] = useState('');

  const [value4, setValue4] = useState('');

  return (
    <div>
      <h3>基础用法</h3>
      <Input value={value1} onChange={val => setValue1(val)} placeholder="请输入..." />

      <h3>多行输入</h3>
      <Input rows={3} value={value2} onChange={val => setValue2(val)} placeholder="请输入..." />

      <h3>高度自适应</h3>
      <Input autoSize value={value3} onChange={val => setValue3(val)} placeholder="请输入..." />

      <h3>限制字数</h3>
      <Input maxLength={20} value={value4} onChange={val => setValue4(val)} placeholder="请输入..." />
    </div>
  );
}
```

# List 列表 [\#](https://chatui.io/components/list#List%20%E5%88%97%E8%A1%A8)

列表容器，可承载文字，图片，按钮操作等，常用于多个连续相同的垂直模块显示。

## 代码演示 [\#](https://chatui.io/components/list#%E4%BB%A3%E7%A0%81%E6%BC%94%E7%A4%BA)

```demo-code jsx
import React from 'react';
import { List, ListItem, Card, CardContent } from '@chatui/core';

export default function () {
  return (
    <div>
      <h3>基础用法</h3>
      <List>
        <ListItem content="内容1" />
        <ListItem content="内容2" />
        <ListItem content="内容3" />
      </List>

      <h3>边框</h3>
      <List bordered>
        <ListItem content="内容1" />
        <ListItem content="内容2" />
        <ListItem content="内容3" />
      </List>

      <h3>右边图标</h3>
      <List>
        <ListItem content="内容1" rightIcon="chevron-right" />
        <ListItem content="内容2" rightIcon="chevron-right" />
        <ListItem content="内容3" rightIcon="chevron-right" />
      </List>

      <h3>渲染成链接</h3>
      <Card>
        <List>
          <ListItem content="内容1" as="a" href="/a" rightIcon="chevron-right" />
          <ListItem content="内容2" as="a" href="/b" rightIcon="chevron-right" />
          <ListItem content="内容3" as="a" href="/c" rightIcon="chevron-right" />
        </List>
      </Card>

      <h3>渲染成按钮</h3>
      <Card>
        <List>
          <ListItem content="内容1" as="button" />
          <ListItem content="内容2" as="button" />
          <ListItem content="内容3" as="button" />
        </List>
      </Card>

      <h3>变体：按钮组</h3>
      <Card>
        <CardContent>
          <List variant="buttons">
            <ListItem content="卖家不退款，我要催退款怎么办" as="button" />
            <ListItem content="怎么申请退款或退货？" as="button" />
            <ListItem content="我要退货，怎么申请？文案很长很长很长很长" as="button" ellipsis />
          </List>
        </CardContent>
      </Card>
    </div>
  );
}
```

# Radio 单选框 [\#](https://chatui.io/components/radio#Radio%20%E5%8D%95%E9%80%89%E6%A1%86)

## 代码演示 [\#](https://chatui.io/components/radio#%E4%BB%A3%E7%A0%81%E6%BC%94%E7%A4%BA)

```demo-code jsx
import React, { useState } from 'react';
import { Radio, RadioGroup } from '@chatui/core';

const options = [\
  { label: '备选项A', value: 'a' },\
  { label: '备选项B', value: 'b' },\
  { label: '备选项C', value: 'c' },\
];

export default function () {
  const [value, setValue] = useState('a');

  function handleChange(val) {
    setValue(val);
  }

  return (
    <div>
      <h3>基础用法</h3>
      <Radio label="备选项A" value="a" />

      <h3>选中状态</h3>
      <Radio label="备选项B" value="b" checked />

      <h3>禁用状态</h3>
      <Radio label="备选项C" value="c" disabled />

      <h3>单选框组</h3>
      <RadioGroup value={value} options={options} onChange={handleChange} />
    </div>
  );
}
```

# 字体 [\#](https://chatui.io/design/font#%E5%AD%97%E4%BD%93)

文字是用户理解产品内容的重要信息媒介，文字的正确设计直接影响用户体验和信息传达的效率。

# 概述 [\#](https://chatui.io/design/font#%E6%A6%82%E8%BF%B0)

**可读**  确保文字易于阅读，字体大小、颜色和背景对比度应该足够清晰。

**层级**  针对不同的信息，文本层级应有不同的字号与字重处理。

**间距**  合适的行间距和字间距有助于文本归类的梳理，提高文本的可读性。

**对齐**  根据内容和布局选择合适的文本对齐方式（左对齐、右对齐、居中对齐、两端对齐）。

**文风**  使用符合淘宝品牌调性和目标用户的文风，确保文本友好、亲切，同时保持专业性。

​

### 字体使用规则 [\#](https://chatui.io/design/font#%E5%AD%97%E4%BD%93%E4%BD%BF%E7%94%A8%E8%A7%84%E5%88%99)

**字体**

在界面字体的运用上淘宝采用系统原生字体，英文：iOS的PingFang和Android的Roboto；中文：iOS的PingFang和Android的Noto（思源黑体）。价格字体统一使用Alibaba Sans 102 V1 Taobao（详见价格字体）优先系统默认字体

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1599570003125-1c53c78b-2bfd-4dc6-81c0-1e7812cc7e6a.png)

### 1. 设计发力点在哪里? 对话设计的机会与限制 [\#](https://chatui.io/design/getting-started#1.%C2%A0%E8%AE%BE%E8%AE%A1%E5%8F%91%E5%8A%9B%E7%82%B9%E5%9C%A8%E5%93%AA%E9%87%8C?%C2%A0%E5%AF%B9%E8%AF%9D%E8%AE%BE%E8%AE%A1%E7%9A%84%E6%9C%BA%E4%BC%9A%E4%B8%8E%E9%99%90%E5%88%B6)

设计点需要满足用户诉求, 业务目标, 和技术的现实可行性, 而机器对话也并不例外。

**设计机会点在用户, 业务与技术的重合点**

例如用户诉求是"全能的对话机器人", 业务诉求是"占领全世界用户", 而目前技术可能在"能力"和"时间"限制上也不能达到, 设计机会点也是不成立的。如果用户诉求是"双11活动那么多, 可以提醒我抢红包吗", 业务目标是"双11拉动新用户增长", 技术也是具备相关的"意图识别, 语意理解, 上下文推理..." 这些对话能力。
**怎么利用好剧本设计, 符合用户的自然对话交互, 这个是设计的机会点。**

### 2. 为什么用户使用对话式, 而不是陈列式界面? 对话式的体验优势 [\#](https://chatui.io/design/getting-started#2.%C2%A0%E4%B8%BA%E4%BB%80%E4%B9%88%E7%94%A8%E6%88%B7%E4%BD%BF%E7%94%A8%E5%AF%B9%E8%AF%9D%E5%BC%8F,%C2%A0%E8%80%8C%E4%B8%8D%E6%98%AF%E9%99%88%E5%88%97%E5%BC%8F%E7%95%8C%E9%9D%A2?%C2%A0%E5%AF%B9%E8%AF%9D%E5%BC%8F%E7%9A%84%E4%BD%93%E9%AA%8C%E4%BC%98%E5%8A%BF)

**理解上下文能力, 主动串联用户任务**

以往的GUI(图形用户界面)产品, 用户如果想出行, 需分别在不同页面寻找汽车预订, 飞机预订, 酒店预订, 行装采购等一系列任务, 寻找成本高, 任务分散。而对话式提供给用户是连续的任务串联体验, 出行是有群体相似的任务模式, 当用户表达了出行, 回答了目的地和时间, 我们便可以串联至完整的对话剧本, 幇助用户预订行程。 以及到了当天主动提示, 管理行程。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1598428125598-a9b0af5a-da35-4a49-8fe5-f56af95a0485.jpeg)

**降低用户认知成本, 可以直达表达想法**

随着业务复杂度的上升, 系统在GUI的表现层级会越变复杂。例如, "我的红包退款去哪了", 在GUI中, 用户需要理解设计师把分类放到"账户, 红包, 退款"等一级或二级类目, 并进行多次尝试, 才能找到最终答案。在对话式中, 用户则可以直接表达想法。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1598428136825-09db52e6-2c42-4b01-9614-59801501e04c.jpeg)

### 3. 有什么对话设计特有注意?绝对路径与概率路径 [\#](https://chatui.io/design/getting-started#3.%C2%A0%E6%9C%89%E4%BB%80%E4%B9%88%E5%AF%B9%E8%AF%9D%E8%AE%BE%E8%AE%A1%E7%89%B9%E6%9C%89%E6%B3%A8%E6%84%8F?%E7%BB%9D%E5%AF%B9%E8%B7%AF%E5%BE%84%E4%B8%8E%E6%A6%82%E7%8E%87%E8%B7%AF%E5%BE%84)

**概念原理: 黑盒子概率路径**

对话在技术上会分成 **工程和算法**, 算法路径基于
**置信度**, 通过样本训练建立相关性路径, 黑盒子的意图连接。而工程是百分之百会按照设计执行, 是因果性逻辑路径, 明确知道上一步和下一步的路径。在他们混合协作下, 有些场景对话是会按因果性工程的绝对路径进行封闭式交互设计, 而意图分析的会以开放式交互自由切换相关性路径, 属于不可预测的自由剧本。

**例子: 用户自由切换路径**

概率路径再举个例子, 用户说"我要买口红", 用户可以先直接看商品, 机器再追问其他决策因子\[品牌/颜色/风格\], 并且用户能自由决定先聊哪个, 或者都不聊, 或者跳到别的对话意图。

**设计注意: 选择合适的交互策略**

绝对路径: 合适 **因果性**
逻辑, 用户不存在可自由表达的可能性, 降低自由输入能, 更好引导用户到达目标。

概率路径: 合适 **相关性**
逻辑, 用户自由表达会更好引发需求, 提升意图进出效率, 更高效引发潜在需求。

## 二、机器对话框架设计 [\#](https://chatui.io/design/getting-started#%E4%BA%8C%E3%80%81%E6%9C%BA%E5%99%A8%E5%AF%B9%E8%AF%9D%E6%A1%86%E6%9E%B6%E8%AE%BE%E8%AE%A1)

### 1. 对话剧本是怎么设计出来的? 基本的用户行为路径 [\#](https://chatui.io/design/getting-started#1.%C2%A0%E5%AF%B9%E8%AF%9D%E5%89%A7%E6%9C%AC%E6%98%AF%E6%80%8E%E4%B9%88%E8%AE%BE%E8%AE%A1%E5%87%BA%E6%9D%A5%E7%9A%84?%C2%A0%E5%9F%BA%E6%9C%AC%E7%9A%84%E7%94%A8%E6%88%B7%E8%A1%8C%E4%B8%BA%E8%B7%AF%E5%BE%84)

我们每一场对话都都在不同颗粒度上的行为设计, 一个机器人是由大量的对话, 剧本, 技能, 垂直域, 由小到大积累形成的。我们基于《HOOKED》《习惯的力量》的广义行为设计原理开始主路径的设计, 以及最终校验是否完成一场完整的对话体验设计。

**触达:** 提供线索, 给用户感知要做些什么, 降低任务难度令用户进入你的对话。

**对话:** 开放对话与封闭对话, 选择合适的对话交互, 分配注意力与耐性, 否则用户将会跳出多轮对话。

**答案:** 答案获取, 结果在前, 理由在后, 情感结尾。内容的结构要先回应用户的问题, 因为用户的耐性与注意力是有限资源, 高效清晰的回复是影响用户体验的关键要素。

**回访:** 类人对话体验, 我们的对话不只是这一次, 像人一样我们会附有记忆。我们会加入更多的个性化, 上下文的能力, 希望最终达至更理想的类人对话体验。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1599570060341-ec275312-f1c9-467f-9819-3428127d9f41.png)

### 2. 对话设计有什么模式? 选择机器人的对话类型 [\#](https://chatui.io/design/getting-started#2.%C2%A0%E5%AF%B9%E8%AF%9D%E8%AE%BE%E8%AE%A1%E6%9C%89%E4%BB%80%E4%B9%88%E6%A8%A1%E5%BC%8F?%C2%A0%E9%80%89%E6%8B%A9%E6%9C%BA%E5%99%A8%E4%BA%BA%E7%9A%84%E5%AF%B9%E8%AF%9D%E7%B1%BB%E5%9E%8B)

我们主要分成2类用户及对应模式。

**服务类-封闭对话模式:** 用户诉求明确, 在 **唯一** 的对话流程进行, 并获取
**唯一** 的答案。

**助理类-开放对话模式:** 用户诉求开放, 能 **自由** 输入, **自由**
选择对话分支, **自由** 进出意图, 并获取多于一个的答案。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1599570337761-be9c722d-3a63-4c2f-bd4d-08ddf9a535ce.png)

### 3. 服务类: 封闭对话, 高效解决, 降低成本, 促成满意度 [\#](https://chatui.io/design/getting-started#3.%C2%A0%E6%9C%8D%E5%8A%A1%E7%B1%BB:%C2%A0%E5%B0%81%E9%97%AD%E5%AF%B9%E8%AF%9D,%C2%A0%E9%AB%98%E6%95%88%E8%A7%A3%E5%86%B3,%C2%A0%E9%99%8D%E4%BD%8E%E6%88%90%E6%9C%AC,%C2%A0%E4%BF%83%E6%88%90%E6%BB%A1%E6%84%8F%E5%BA%A6)

用户能高效获取答案, 并由机器最大化解决, 能促进业务人工成本下降。所以, 服务场景我们会希望引导最大化达到答案的机会率。例如, 关闭自由输入, 提升用户专注力, 集中在当前对话, 降低对话过程跳出率。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1599570157035-024de878-e156-4cb4-be48-dd831ef62aeb.png)

**3.1 机器高效对话, 降低人力成本**

用户诉求明确, 业务解决方案标准化, 我们在大部份对话场景会使用封闭交互, **关闭自由输入, 防止对话过程中出现意图跳失**。服务场景, 我们需要帮助用户有效获取答案。

例如, "我的红包退款去哪了", 用户需要明确, 选择订单, 确定问题, 机器才能有效回答。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1599570202925-c7fb9c82-dae9-4ffb-9a5c-40a0c7e7c097.png)

**平台上我们有什么服务领域对话可以参考?**

ChatUI是建基于阿里巴巴电商平台生态的背景, 所以电商, 服务进度, 退款, 维权, 出行, 饮食,等 均有一定沉淀, 以帮助行业降低门槛, 加速发展, 共建对话体验增长。 而在不同的路径上我们有对应的模块, 帮助用户进行交互。

**3.1.1 触达: 未问先答, 积极主动**

用户可能由不同入口进来, 主要分成两大类\*\*,\*\* 自然触达和推送触达。

**1. 自然触达: 用户目的不明确, 主动打开机器人**

提问有门槛, 用户需要我们通过界面组件引导, 提供行为线索, 降低提问门槛, 促成对话。
我们常用的设计组件, 例如: 简报, 技能卡, 快捷短语, 语音输入, 工具。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1581048112673-f3623df7-cb21-4ce8-a353-ef3c8a92ee54.png)

整体布局, 我们会依阅读的习惯进行布局, 首焦是用户相关度较高的信息内容, 往下是类目相关的内容, 颗粒度由上至下分布。

以下是具体各细节组件的作用:

1. 话术: 利用对话说明现在的情况

2. 待办: 有个人事件信息更新时出现提示

3. 简报: 通过算法及运营规则, 预测用户可能咨询的问题

4. 技能卡: 引导用户进入意图类目

5. 快捷短语: 降低用户输入成本

6. 语音输入: 降低用户键盘输入成本

7. 工具: 帮助用户输入富交互内容, 例如: 订单, 商品, 图片, 视频

8. 技能中心: 过了首屏后, 用户可以找到技能, 并达成意图发送

**2. 推送触达: 机器主动找用户对话**

**及时主动,** 用户对服务不满意的因素中包括机器不及时不主动, 因此机器主动对话也是促进满意度的关键抓手之一。

我们应用的场景, 例如, 由订单售后会带着"订单"(事件), "卖家不发货"(对话主题), 进来。对应不同的情况我们会利用界面组件提供线索, 引导操作, 促成对话, 达成用户的期待。我们会通过组件, 提供线索: 主动服务, 障碍预测。

**3.1.2 对话: 封闭对话, 绝对路径, 引导用户高效操作**

在封闭路径场景, 我们会通过关闭自由输入, 加深背景层, 突出当前任务操作层, 促成用户在场景中最大完成率。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1581048216778-a5bfc442-5b95-4326-8a5d-deb25f3e1bed.png)

我们常用的组件:

1. 订单选择器: 咨询服务主要是基于订单(事件)进行。

2. 半层选项: 服务标准流程问题(SOP)是由后台的绝对路径配置, 用户需要较高注意选择并决策。

3. 进度展示: 用户部分的对话并不能获取最终服务结束, 我们会透出进度安抚用户, 或提供催促行动点幇助用户表达焦虑, 以促成过程中的满意度达成。

4. 表单: 复杂问题可能会用到表单, 进行跨时间的人工介入回复及解决。

**3.1.3 答案: 高效直白, 结果在前, 理由在后, 情感结尾**

**内容框架: 结果在前, 理由在后, 情感结尾。**

因为用户在前面购物遇到问题已经不满意, 好不容易才找到我们, 再进行了多轮对话, 才能看到答案, 用户情绪焦虑, 不耐烦的负面情绪都在这里了。所以高效的表达用户最需要答案, 节省用户的时间。结尾提供情感化内容, 基于\*\*"峰终定律"\*\*将会进一步强化用户的好感记忆。

**有什么是对话特有的设计手段?**

情感计算, 机器人基于即时对话(IM)沟通, 我们可以和人对话一样用表情emoji及GIF表达情感, 而我们有实验组对照发现emoji在答案中出现能
**降低不满意度**。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1581048267896-2fdca120-d001-40ff-8e48-975cb79dbfa7.png)

**3.1.4 回访, 个性化记忆, 类人的对话体验**

记忆, 对话的体验像人和人沟通类似, 当用户上次问题未结束, 机器会进跟确认。用户上滑是历史对话, 以及会有相应的未读提示。每次首进会有相关的个性化首页信息预测。

**障碍预测**: 当用户上次问题未结束, 机器会进跟确认

**历史消息**: 用户上滑是历史对话

**首页**: 每次首进会有相关的个性化首页信息预测

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1581048331870-152d5f41-3a85-443a-8548-24feb5fcd8a0.png)

**3.2 人工服务, 人机协同, 解决用户断点**

**触达**: 入口

**对话**: 开场, 问答

**答案**: 结束

**回访**: 进度

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1599570242960-6a27ceeb-b6f8-447c-9838-863bcf730967.png)

### 1. 触达: 算法动态入口, 资源与满意度的取舍 [\#](https://chatui.io/design/getting-started#1.%C2%A0%E8%A7%A6%E8%BE%BE:%C2%A0%E7%AE%97%E6%B3%95%E5%8A%A8%E6%80%81%E5%85%A5%E5%8F%A3,%C2%A0%E8%B5%84%E6%BA%90%E4%B8%8E%E6%BB%A1%E6%84%8F%E5%BA%A6%E7%9A%84%E5%8F%96%E8%88%8D)

入口: 主要由快捷短语由算法计算什么时机合适给出人工入口, 以及哪类人工入口, 包括电话直拨, 电话回拨, 网络电话, 在线客服。如果你有需要, 也可以用卡片引导用户转人工。

**没有黄边**: 用户有可能没注意到人工入口, 在找不到人工中约占7%

**有黄边**: 用户点击率较有黄边差值2%, 文案差值2%, 建议: "联系人工客服"

**可视化卡片**: 用户的点击率较高, 目前未有人工业务使用

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1581048686591-1fd946e5-e9b8-4dbd-9740-74b9d41c2771.png)

### 2.对话: 沟通效率与资源优化 [\#](https://chatui.io/design/getting-started#2.%E5%AF%B9%E8%AF%9D:%C2%A0%E6%B2%9F%E9%80%9A%E6%95%88%E7%8E%87%E4%B8%8E%E8%B5%84%E6%BA%90%E4%BC%98%E5%8C%96)

在开场时, 系统会发出人工的名片, 给用户感知人工区隔, 用户也可以由名片中获取人工的简要信息, 快速建立互信, 提升沟通效率。

对话过程中, 快捷短语会置换成常用的富媒体工具, 提升沟通效率。及结束服务, 促进系统中人工资源利用率提升, 更多有需要的用户能获取人工资源。

**名片**: 感知人工与机器区隔, 建立信任

**快捷工具**: 常用工具透出; 结束人工意图;

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1581048742132-f9baa5a8-5513-4091-9d28-5e49a43d4052.png)

### 3.答案: 结束, 完整的体验闭环, 满意度监测基制 [\#](https://chatui.io/design/getting-started#3.%E7%AD%94%E6%A1%88:%C2%A0%E7%BB%93%E6%9D%9F,%C2%A0%E5%AE%8C%E6%95%B4%E7%9A%84%E4%BD%93%E9%AA%8C%E9%97%AD%E7%8E%AF,%C2%A0%E6%BB%A1%E6%84%8F%E5%BA%A6%E7%9B%91%E6%B5%8B%E5%9F%BA%E5%88%B6)

用户结束服务后我们会追问人工服务的表现, 形成体验监测闭环, 促进体验的自我完善。

### 为什么需要对话式设计？前沿技术，到无处不在的对话 [\#](https://chatui.io/design/introduce#%E4%B8%BA%E4%BB%80%E4%B9%88%E9%9C%80%E8%A6%81%E5%AF%B9%E8%AF%9D%E5%BC%8F%E8%AE%BE%E8%AE%A1%EF%BC%9F%E5%89%8D%E6%B2%BF%E6%8A%80%E6%9C%AF%EF%BC%8C%E5%88%B0%E6%97%A0%E5%A4%84%E4%B8%8D%E5%9C%A8%E7%9A%84%E5%AF%B9%E8%AF%9D)

随着人工智能的技术成长, 云端算力的不断提高，由最初由先进国家主导的对话式交互, 例如Google助理、Apple Siri、Microsoft Cortana、IBM Waston, 发展至今, 国内已全面普及, 无所不在, 不知不觉中走进每个人的生活, 例如智能家居的天猫精灵、小爱同学、百度小度、华为小艺等。以及, 不同行业的日常生活场境, 银行、通信、出行、电商、餐飲、保险、金融、政务等。背后遂渐由人力劳动，转移至技术劳动，解决更多的人力对话诉求。

### 什么是对话式设计? 合适对话的用户场景​ [\#](https://chatui.io/design/introduce#%E4%BB%80%E4%B9%88%E6%98%AF%E5%AF%B9%E8%AF%9D%E5%BC%8F%E8%AE%BE%E8%AE%A1?%C2%A0%E5%90%88%E9%80%82%E5%AF%B9%E8%AF%9D%E7%9A%84%E7%94%A8%E6%88%B7%E5%9C%BA%E6%99%AF%E2%80%8B)

**CUI**(Conversation User Interface, 会话式用户介面)
**与GUI**(Graphical User Interface, 图形用户介面) **对比。**
并不是所有场景都合适使用CUI会话式用户介面, CUI当中我们也会混合使用到GUI图形用户介面, 或者更多模态的方式, 促进用户与机器之间, 高效沟通。情况就像人和人的沟通，也是会通过图片, 文字, 动画, 表达想法, 例如我们制作的PPT, 视频和文章辅助沟通交流一个道理。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1600152969003-4f56c3e7-7709-4c1e-8b8e-284cc7ff35f5.jpeg)

**理解上下文能力, 主动串联用户任务。**
以往的GUI产品, 用户如果想出行, 需分别在不同页面寻找汽车预订, 飞机预订, 酒店预订, 行装采购等一系列任务, 寻找成本高, 任务分散。而对话式提供给用户是连续的任务串联体验, 出行是有群体相似的任务模式, 当用户表达了出行, 回答了目的地和时间, 我们便可以串联至完整的对话剧本, 帮助用户预订行程。 以及到了当天主动提示, 管理行程。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1600153127512-729c1af3-4688-4727-8a34-672a011999ef.jpeg)

**降低用户认知成本, 可以直达表达想法。**
随着业务复杂度的上升, 系统在GUI的表现层级会越变复杂。例如, "我的红包退款去哪了", 在GUI中, 用户需要理解设计师把分类放到"账户, 红包, 退款"等一级或二级类目, 并进行多次尝试, 才能找到最终答案。在对话式中, 用户则可以直接表达想法。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1600153137921-97b5e8c9-066a-40dd-9595-ef7cc72a135c.jpeg)

### ChatUI的对话式设计原则 [\#](https://chatui.io/design/introduce#ChatUI%E7%9A%84%E5%AF%B9%E8%AF%9D%E5%BC%8F%E8%AE%BE%E8%AE%A1%E5%8E%9F%E5%88%99)

ChatUI是基于人工智能的对话式机器人CUI组件, 最终愿景是达至"类人"自然交互。通过一场场的场景剧本设计及机器训练, 机器由话术与界面多模组合与用户进行互动沟通。

我们经过设计共创由大量竞品分析及思辨, 最终选择了四个关键体验目标, 能有效促进用户对机器智能的感知。

- **主动**: 用户还没想起, 机器能预知用户所需。

- **高效**: 用户使用流顺, 符合自然预期, 有效进行互动。

- **情感**: 用户有情感引导, 情感互动, 产生心理偏好或习惯，达至自然回访。

- **沉浸**: 用户能通过视觉, 互动及物理传感器等触点, 感受智能的新鲜感，引发交互意愿。

# 圆角 [\#](https://chatui.io/design/radius#%E5%9C%86%E8%A7%92)

圆角是单元模块中矩形四周角变为弧形的过渡，提升模块灵活性的同时给用户界面带来柔和的感受。

# 概述 [\#](https://chatui.io/design/radius#%E6%A6%82%E8%BF%B0)

**梯度**  在一套设计系统中使用统一的圆角大小可以增强界面的一致性和品牌识别度，我们将不同的产品层级模块圆角收敛为4种分层单位数值。

统一圆角的通用运用为核心的如下2种，弱化套角的层级表达，让产品体验更加工整 · 有序 · 中性。

![image](https://chat-ui.oss-cn-hangzhou.aliyuncs.com/design-imgs/1739850554207-201d835d-41bc-41bf-9a2b-8a1b7d0f35ca.png)

### 圆角使用规则 [\#](https://chatui.io/design/radius#%E5%9C%86%E8%A7%92%E4%BD%BF%E7%94%A8%E8%A7%84%E5%88%99)

通用的圆角数值为12px，用于面板卡片等场景，如浮层、模块、控件、媒体等基础结构型产品模块圆角。

特殊的圆角数值为8px，用于极小组件场景，如标签、气泡、角标等标注结构型产品组件圆角。

全圆角的数值为9999px，用于胶囊型、全弧形组件，如开关、勾选等场景。

无圆角的数值为0px，用于全背景及固定模块，如全局背景、底栏背景、导航栏背景等模块场景。

如果你需要配置一个独立页面，限宽窗口可以帮助用户有效利用屏幕宽度。我们提供了默认底图，降低你的配置成本，你也可以自己配置符合你品牌需求的底图增强氛围。

合适中国另开新页, 有效宽度控制眼动及操作距离, 促成沉浸复杂咨询服务。

有一个设计细节可能你需要了解一下：

**背景**：兜底的图片为用户正在使用电脑进行输入，支持配置成品牌更合适图片, 促成各业务目标。

## B: 全屏窗口 [\#](https://chatui.io/design/responsive#B:%C2%A0%E5%85%A8%E5%B1%8F%E7%AA%97%E5%8F%A3)

如果你的网站需要有自己的顶通导航返回主站，全屏窗口可能更合适你进行二次开发。

合适国际网站当前刷新, 网站有顶通导航, 促成沉浸复杂咨询服务。

有二个设计细节可能你需要了解一下：

# 间距 [\#](https://chatui.io/design/space#%E9%97%B4%E8%B7%9D)

间距是指组件之间的间距，合理的间距能够使页面有清晰、实用的布局。

# 概述 [\#](https://chatui.io/design/space#%E6%A6%82%E8%BF%B0)

**计算单元**  以4px为基础单位，采用倍数拓展，如6px、8px、12px、16px…

**非Token组件**  考虑到系统兼容性问题，间距暂未作为常量Token进行代码化，当前主要作为设计规范指导界面方案的输出与还原，在增强视觉层次的同时，提高信息可读性，保障用户体验一致性。

# 定制主题 [\#](https://chatui.io/docs/customize-theme#%E5%AE%9A%E5%88%B6%E4%B8%BB%E9%A2%98)

ChatUI 提供了一套默认主题，CSS 命名采用类 BEM
的风格，方便使用者覆盖样式。如果你想完全替换主题色或者其他样式，可以使用下面提供的方法。

## 样式变量 [\#](https://chatui.io/docs/customize-theme#%E6%A0%B7%E5%BC%8F%E5%8F%98%E9%87%8F)

ChatUI 对部分经常有定制需求的属性用了
[CSS 变量](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_cascading_variables/Using_CSS_custom_properties)
抽象，并大致分成了主题色、文字色、填充色、线条色、功能色等几类，你可以根据需求进行相应调整。

下面是一些基本的样式变量，更多的可以查看
[root.less](https://github.com/alibaba/ChatUI/blob/master/src/styles/root.less)
。

```demo-code less
:root {
  // Brand 品牌色
  --brand-1: #ff6200;
  --brand-2: #ffb300;
  --brand-3: #ffe0cc;

  // Text 文字色
  --color-text-1: var(--gray-1);
  --color-text-2: var(--gray-2);
  --color-text-3: var(--gray-3);

  // Fill 填充色
  --color-fill-1: var(--white);
  --color-fill-2: var(--gray-8);

  // Line 线条色
  --color-line-1: var(--gray-5);
  --color-line-2: var(--gray-4);

  // Functional 功能色
  --highlight-1: var(--brand-1);
  --highlight-2: var(--brand-2);
  --color-warning: var(--red);
  --color-success: var(--green);
  --color-link: var(--link-color);

  // FontSize 字号
  --font-size-xs: 0.75rem;
  --font-size-sm: 0.875rem;
  --font-size-md: 1rem;
  --font-size-lg: 1.125rem;

  // Radius 圆角
  --radius-sm: 4px;
  --radius-md: 6px;
}
```

## 定制方法 [\#](https://chatui.io/docs/customize-theme#%E5%AE%9A%E5%88%B6%E6%96%B9%E6%B3%95)

```demo-code less
import '@chatui/core/dist/index.css';
```

# 国际化 [\#](https://chatui.io/docs/i18n#%E5%9B%BD%E9%99%85%E5%8C%96)

## Chat [\#](https://chatui.io/docs/i18n#Chat)

可通过 `locale` 属性切换当前使用的语言。

```demo-code jsx
import Chat from '@chatui/core';

return <Chat locale="zh-CN" />;
```

## LocaleProvider [\#](https://chatui.io/docs/i18n#LocaleProvider)

同时 ChatUI 提供了 `<LocaleProvider>` 组件用于配置国际化文案。

```demo-code jsx
import React from 'react';
import { LocaleProvider } from '@chatui/core';

const locales = {
  text: '文本',
};

return (
  <LocaleProvider locales={locales}>
    <MyCard />
  </LocaleProvider>
);
```

```demo-code jsx
// MyCard
import React from 'react';
import { useLocale } from '@chatui/core';

export default function MyCard({ data = {}, ctx }) {
  const { trans } = useLocale();
  return <Button>{trans('text')}</Button>;
}
```

## 渲染界面 [\#](https://chatui.io/docs/quick-start#%E6%B8%B2%E6%9F%93%E7%95%8C%E9%9D%A2)

通过 `<Chat>` 组件渲染出对话界面：

```demo-code jsx
const App = () => {
  const { messages, appendMsg, setTyping } = useMessages([]);

  function handleSend(type, val) {
    if (type === 'text' && val.trim()) {
      appendMsg({
        type: 'text',
        content: { text: val },
        position: 'right',
      });

      setTyping(true);

      setTimeout(() => {
        appendMsg({
          type: 'text',
          content: { text: 'Bala bala' },
        });
      }, 1000);
    }
  }

  function renderMessageContent(msg) {
    const { content } = msg;
    return <Bubble content={content.text} />;
  }

  return (
    <Chat
      navbar={{ title: '智能助理' }}
      messages={messages}
      renderMessageContent={renderMessageContent}
      onSend={handleSend}
    />
  );
};
```

更多参数可查看 [Chat 对话容器](https://chatui.io/components/chat)。
